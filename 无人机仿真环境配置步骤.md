# 无人机仿真平台搭建和仿真步骤

## 1 前面的话

搭建出一个能够使用的仿真平台是很容易的。

使用Ubuntu20.04.6desktop-amd64作为操作系统。工具链为ROS2-Gazebo-PX4。

配置的流程参照如下链接，并在一些地方作出了补充

### px4是什么

PX4 是一款**开源的自动驾驶仪软件**，专为无人机、无人车、机器人及其他自动驾驶系统设计。它由社区驱动，遵循 BSD 许可证，允许用户自由使用、修改和分发代码，广泛应用于学术研究、工业开发和业余爱好者项目。

> BSD许可证是一种开源协议，如果使用了包含该开源项目的源代码或者二进制文件，需要在发布的项目中必须包含源代码中的BSD协议；在项目发行或推广时，不能使用开源代码的作者/机构名字和原来产品的名字做市场推广。

px4在使用方面包含了丰富的飞行控制算法，对各种硬件都有适配；在开发和设计方面，它具有模块化的设计，还提供了一套仿真工具，同时还有对外部设备的通信协议。

### Gazebo是什么

**Gazebo** 是一款**开源的机器人仿真平台**，专为复杂物理环境中的机器人建模、算法测试和场景模拟而设计。它能够高精度模拟真实世界的物理特性（如重力、摩擦力、光照等），常与机器人操作系统 **ROS/ROS 2** 结合使用，是机器人开发、自动驾驶、无人机控制等领域的重要工具。

### ROS和ROS2是什么

ROS系统对机器人硬件作出了抽象，可以理解为*“操控机器人的编程语言”*，

### 为什么要使用这个技术路线

ROS/ROS2与PX4和Gazebo结合，可以构建从仿真到实机的全部流程，大幅度地降低了开发、部署和集成地难度。



## 2 如何安装和使用虚拟机

到国内各个大学的开源镜像网站中找到对应版本的ubuntu系统下载。

VMware是安装虚拟机的工具，在2024年开放了17以后Pro版本的个人免费使用，2025年4月，官网还没修好，找了其他的下载地址。如果链接不可用了，请自行到官网下载。

> [GitHub - yanghao5/VM-download: VM unofficial download](https://github.com/yanghao5/VM-download)

在第一次对虚拟机分配内存和磁盘空间的时候，多分配一点，推荐内存16+G，磁盘空间40+G。

安装好了虚拟机后，不要对ubuntu系统的桌面进行美化，可以添加中文输入法，但是不能修改系统语言。

更改Ubuntu系统的软件源下载地址。由于系统默认的软件源下载网站大多在国外，为了节约时间，将软件包管理器的下载地址替换成国内镜像源。

**每完成一个工作节点请务必保存一个快照**（就是备份，避免虚拟机崩溃后又要从头开始，避免仿真环境配置失败后残留卸载麻烦）。需要注意的是，虚拟机快照将消耗较大的磁盘空间，需要在安装时选择较大的主机磁盘经行安装。

2025年4月，安装ubuntu20虚拟机后，系统会提示是否升级到ubuntu22，可以更新，但是不同版本的ubuntu系统对应不同的ROS2版本，需自行确定。

## 4 如何安装PX4

px4一般需要将全部源代码下载下来到本地编译，来确保不会因缺少依赖而影响后续的工作。该项目体积较大，可以采取各种手段进行加速。各种方案如下：

- 科学上网，加速Github访问速度。最快的办法。
- 更改host，这一步可以手动完成，不过要备份；也可以借助第三方工具(加速器)来完成。如果更改虚拟机host，需要配置虚拟机网络的连接方式；如果更改主机的host，需要配置虚拟机使用主机的网络。比较稳定，但是速度没有特别快。
- 查询Github加速代理(推荐)。如果是简单地使用，会在下载其他依赖时降低速度，目前还不知道如何解决。建议和更改host方法一起使用，增加连接稳定性。
- 寻找国内的资源，下载到本地后手动编译，但可能会出现版本问题。

```bash
mkdir PX4
cd  PX4
git clone https://github.com/PX4/PX4-Autopilot.git --recursive 

bash ./PX4-Autopilot/Tools/setup/ubuntu.sh
cd PX4-Autopilot/
make px4_sitl
```

在对Github的访问加速后，采取上述操作。一般在根目录下进行。如果在其他目录经行上述操作，请注意不同安装教程的相对路径关系。

```bash
bash ./PX4-Autopilot/Tools/setup/ubuntu.sh
```

上面的这行代码的作用就是为了安装一些依赖，在执行这步命令时，确保已经更换了apt包管理器的软件源和python的第三方库软件源。

### **安装PX4的子模块**

上面的步骤没有下载PX4的子模块，在`PX4-Autopilot/.gitmodules`中规定了子模块的下载方式。为了保证下载速度，需要打开该文件，将其中的链接按加速代理的方式进行修改后保存。修改后执行命令

```bash
git submodule update --init --recursive
```

不同的项目分支子模块有所不同，如果需要更改分支，执行以下命令后再执行以上命令

```bash
git checkout v1.14.0
```

在上述安装过程中如出现了缺少某些库导致失败，直接手动安装后再次运行。

## 7 如何安装多个ros版本

对于需要同时使用ROS1和ROS2的情况，需要安装多个ros。一般来说安装的方式与单独安装一致，但是一般在一个shell中只能启动一个ros版本。

安装了其他版本后需要在在根目录下`.bashrc`的文件添加环境变量设置

```bash
echo "ros noetic(1) or ros2 foxy(2)?"
read edition
if [ "$edition" -eq "1" ];then
  source /opt/ros/noetic/setup.bash
  echo "ROS Noetic env loaded."
else
  source /opt/ros/foxy/setup.bash
  echo "ROS2 foxy env loaded."
fi
```

对于不同的ros版本上述配置文件需要在对应位置修改，需要简单了解后进行修改工作。

## 8 ROS2 创建一个工作空间并运行节点

### 1)创建ROS2 workshop 工作空间

```bash
# 创建~/dev_ws/src 没有则新建
# dev_ws develop workshop 工作空间
# src source code 源代码
mkdir -p ~/dev_ws/src
```

这一步只是为了便于项目管理，名字可以根据喜好而定。

### 2)创建节点

```bash
cd ~/dev_ws/src
# --build-type ament_cmake 指定了编译方式,
# <package_name> 在具体操作时，将其替换为具体的包名

```

在工作空间下的src目录下创建节点

### 3)编译节点

```bash
cd ~/dev_ws
# 编译工作空间下的所有节点

```

在工作空间目录下经行编译。

### 5)如何解决cmake版本问题

在由于在之前的仿真环境的配置中没有关注cmake的版本控制问题。在具体编写节点的时候发生了错误。

首先ubuntu20系统默认的cmake版本为3.16。在px4编译过程中会提示版本过低的报错，尝试替换为cmake3.3版本后成功。

在编译ros2节点时发现最低cmake版本要求为3.5，在替换后px4竟然无法正确编译，即无法正确启动px4-gazebo仿真环境，提示不存在的cmake文件。

为了解决这个问题。

- 尝试采用cmake官方的版本控制器来使用cmake，解决统一的cmake问题
- ros2节点的编译也许可以采用别的方法

2025年4月21日，cmake安装路径有优先级的问题。第一次尝试出错。

##### 最简单的安装方法

```bash
sudo apt install cmake
```

采用系统apt工具安装，一般安装的是3.16，版本太低，不行。不能用多个版本。

##### 下载源码编译安装

该安装方法能够完全控制版本。但是需要注意几点

- cmake官方网站没有cmake的源码下载，只有二进制文件。如果需要下载源代码可以从清华镜像或者github上下载。
- 不能多个版本同时使用

```bash
# 解压压缩包, 运行时将cmake-3.24.1.tar.gz 替换成对应的文件名
# 源码安装的压缩包体积很小，如果你的压缩包文件有个几十M, 合理推测是二进制文件而非源代码,步骤略有不同
tar -zxvf cmake-3.24.1.tar.gz

# 只有从源代码安装才有此步骤
./bootstrap

# 安装
make
sudo make install

# 检查版本信息
cmake -version
# 查看路径
which cmake
```

##### ppa 安装

PPA 软件仓库包，将包含特定版本cmake的软件包加入系统环境中，使其能够安装不同版本的cmake。

通过以下步骤安装最新版cmake

```bash
wget -O - https://apt.kitware.com/keys/kitware-archive-latest.asc 2>/dev/null | sudo apt-key add -

sudo apt-add-repository 'deb https://apt.kitware.com/ubuntu/ bionic main'
sudo apt-get update

sudo apt install cmake
```

如需安装其他版本

```bash
apt list -a cmake
# 上一个操作会列出版号，复制粘贴到下面即可
sudo apt install cmake=3.27.7-0kitware1ubuntu22.04.1
```

通过**`update-alternatives`**管理版本

不同的安装方式下面的参数不相同，请正确指定cmake的调用和安装路径

最后的数字是默认的优先级，数字越大优先级越高

```bash
sudo update-alternatives --install /usr/bin/cmake cmake /usr/bin/cmake3.27 100
sudo update-alternatives --install /usr/bin/cmake cmake /usr/bin/cmake3.16 80
```

切换版本

```bash
sudo update-alternatives --config cmake
```

实测后不行达不到要求。

##### 下载源码进行安装

下载cmake源码，注意更换cmake的版本名称

```bash
wget https://cmake.org/files/v3.22/cmake-3.22.1.tar.gz
```

解压

```bash
tar -xvzf cmake-3.22.1.tar.gz
```

安装依赖，可选

```bash
sudo apt-get install libssl-dev
```

切换到解压项目的根目录下

```bash
cd cmake-3.22.1
chmod 777 ./configure
./configure
```

编译

```bash
make
```

安装

```bash
sudo make install
```

```bash
# 如果只要安装这一个cmake 这样写不会有问题, 如果要实现多个camke版本管理，建议仔细理解该命令的相关参数
sudo update-alternatives --install /usr/bin/cmake cmake /usr/local/bin/cmake 1 --force
```

2025.4.22。px4编译时的cmake版本需要大于3.1小于3.5。而ros2自动创建的功能包最小版本为3.5，有很大的概率不会重合，尝试在这个基础上使用多版本cmake。如果过于麻烦，将会尝试cmake3.5是否能完全兼容。如果完全按照上面的做法会有点麻烦，但能用。可能会出现的问题

- 上述方法必须在信任的软件源下载源代码，由于安装过程中给予了过高的权限，可能带来的安全问题
- 不同的版本在同一个目录下安装，每次都要重新安装然后才能使用。如果在不同的目录下安装，就可以直接切换

预计后续继续修改方法。

修改命令，指定安装路径，与其他cmake版本区分。

```bash
./configure --prefix=/opt/cmake-3.30.0
```

2025.4.22，下面的方法还没验证

```bash
# 解压源码包
tar -xzf software.tar.gz
cd software

# 1. 赋予 configure 可执行权限（仅在无权限时执行）
# 生成一个configure 可执行文件
chmod +x ./configure

# 2. 运行配置脚本
./configure --prefix=/opt/software-version

# 3. 编译源码
make

# 4. 安装软件
sudo make install
```

值得注意的是接下来的步骤

**--install 命令有四个参数**

`update-alternatives --install`为调用命令时搜索的路径

`cmake`为调用时的命令

`/opt/software-version/bin/cmake`为实际安装路径，注意修改

`100` 为优先级级。默认将使用优先级最高的cmake，数字越大优先级越高，

```bash
sudo update-alternatives --install /usr/bin/cmake cmake /opt/software-version/bin/cmake  100
```
